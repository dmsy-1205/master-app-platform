# Known Issues

## 현재 확인 필요 항목

### 1. 기존 로컬 로그인 세션 잔존 가능성

STEP8 v2부터 브라우저 세션 유지 방식을 적용했지만 이전 버전에서 이미 로컬 로그인 세션이 남아 있는 브라우저는 한 번 로그아웃해야 새 방식이 확실히 적용됩니다.

### 2. Firebase Rules 미적용

현재는 개발 단계이므로 클라이언트 기능 검증이 우선입니다.
보안 규칙은 STEP9에서 적용해야 합니다.

### 3. 관리자 수동 등록 도구

`현재 계정을 관리자 등록` 버튼은 개발 단계 테스트용입니다.
최종 배포 전에는 관리자 등록 방식을 별도 보안 절차로 분리해야 합니다.

### 4. 내부 라우터 실행 방식

`router` 방식은 같은 도메인 또는 CORS 허용 HTML에 적합합니다.
외부 Netlify 앱은 보통 `newTab` 실행이 더 안정적입니다.

### 5. 한 파일 중심 구조

STEP8 v2에서 화면 노출 구조는 정리했지만 아직 프로젝트는 단일 `index.html` 중심 구조입니다.
STEP9 이후에는 필요에 따라 파일 구조 분리를 검토할 수 있습니다.
