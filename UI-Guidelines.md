# UI-Guidelines.md

## 디자인 목표
전문 SaaS 플랫폼처럼 보이는 깔끔하고 신뢰감 있는 UI를 유지한다.

## 핵심 규칙
- 로그인 전 화면은 서비스 소개와 인증에 집중한다.
- 로그인 후 화면은 사용자 Workspace와 관리자 Center를 분리한다.
- 앱 목록은 App Store형 카드 UI를 기본으로 한다.
- 개발 테스트 기능은 일반 사용자에게 노출하지 않는다.
- 색상은 블루/바이올렛/화이트 중심으로 유지한다.
- 버튼과 카드에는 충분한 여백과 둥근 모서리를 적용한다.


## STEP9-v4 UI 규칙
- 상단 검색은 앱 탐색 중심으로 사용한다.
- 알림과 프로필은 우측 상단 패널로 제공한다.
- 최근 활동은 Dashboard와 활동 로그 화면에서 동시에 확인한다.
- 즐겨찾기는 앱 카드 안에서 즉시 토글한다.


## STEP9-v4 UI 기준
- App Store 카드의 주요 CTA는 실행 버튼으로 둔다.
- 즐겨찾기는 보조 기능이므로 작은 아이콘형 버튼으로 표현한다.
- 관리자 위험 작업 삭제는 빨간색 계열 버튼과 확인창을 함께 사용한다.
