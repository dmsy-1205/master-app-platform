# Master App Platform - STEP 8 v2

## 현재 버전

`master-app-platform-step08-v2`

## 프로젝트 개요

Master App Platform은 여러 개의 웹 애플리케이션을 하나의 플랫폼에서 운영하기 위한 통합 플랫폼입니다.

이번 버전은 STEP8 앱 연결 기능을 보완하면서 개발 단계에서 발견된 구조 문제를 함께 정리한 안정화 버전입니다.

## STEP8 v2 핵심 목표

- 한 페이지에 모든 기능이 무조건 노출되던 구조를 정리
- 로그인 전 화면과 로그인 후 화면 분리
- 사용자 영역과 관리자 영역 분리
- 개발 테스트 기능을 접기/펼치기 도구로 이동
- 관리자 권한이 재로그인 후에도 유지되도록 보강
- 로그인 유지 방식을 브라우저 세션 기준으로 조정
- STEP8 앱 실행 연결 기능을 사용자 Dashboard 중심으로 정리

## 이번 버전에서 수정한 내용

### 1. 화면 구조 정리

기존에는 `index.html` 한 화면에 다음 기능이 모두 노출되었습니다.

- 회원가입
- 로그인
- 로그인 상태
- DB 테스트
- 관리자 권한 테스트
- 사용자 Dashboard
- 회원 신청 시스템
- 관리자 승인 대시보드
- 앱 등록 관리
- 라우팅 테이블
- 앱 실행 화면

STEP8 v2에서는 다음 구조로 정리했습니다.

- 로그인 전: 회원가입 로그인만 표시
- 로그인 후: 로그인 상태 사용자 Dashboard 회원 신청 앱 실행 화면 표시
- 관리자 권한 확인 후: 관리자 승인 대시보드 앱 등록 관리 라우팅 테이블 표시
- 개발 테스트 도구: 접기/펼치기 영역으로 이동

### 2. 로그인 유지 방식 개선

Firebase Authentication의 기본 로컬 유지 방식 대신 브라우저 세션 유지 방식을 적용했습니다.

- 로그인 상태는 현재 브라우저 세션에서 유지됩니다.
- 창을 완전히 닫은 뒤 새로 접속하면 다시 로그인이 필요합니다.
- 기존에 이미 로컬 유지된 세션은 한 번 로그아웃하면 새 방식으로 정리됩니다.

### 3. 관리자 권한 유지 문제 수정

기존 관리자 권한 확인이 `admins/{uid}` 기준에 강하게 의존해 재로그인 후 일반 사용자처럼 보일 수 있던 부분을 보강했습니다.

관리자 권한은 다음 두 경로를 함께 확인합니다.

```text
admins/{uid} === true
users/{uid}/role === "admin"
```

관리자 등록 시 두 경로를 함께 동기화합니다.

### 4. STEP8 앱 실행 보완

사용자 Dashboard에서 활성화된 앱을 카드 형태로 표시하고 실행할 수 있습니다.

지원하는 실행 방식은 다음과 같습니다.

- `router`: 플랫폼 내부 Runtime View에서 실행
- `newTab`: 새 탭으로 실행
- `sameTab`: 현재 창에서 이동

앱 실행 시 다음 데이터가 Firebase에 저장됩니다.

```text
apps/{appId}/runCount
apps/{appId}/lastRunAt
apps/{appId}/lastRunBy
users/{uid}/lastAppLaunch
appRunLogs/{uid}/{appId}
```

관리자는 승인 상태와 별개로 테스트 목적으로 앱 목록과 실행 기능을 확인할 수 있습니다.

## 테스트 방법

1. ZIP 압축을 풉니다.
2. GitHub 기존 프로젝트에 덮어쓰기 업로드합니다.
3. Netlify 자동 배포 완료 후 접속합니다.
4. 로그인 전에는 회원가입 로그인만 보이는지 확인합니다.
5. 로그인 후 사용자 Dashboard 회원 신청 Runtime View가 보이는지 확인합니다.
6. 개발 테스트 도구가 접기/펼치기 영역에 있는지 확인합니다.
7. 현재 계정을 관리자 등록합니다.
8. 로그아웃 후 다시 로그인하여 관리자 영역이 자동으로 표시되는지 확인합니다.
9. 앱 등록 후 Dashboard에서 실행 버튼이 작동하는지 확인합니다.
10. 실행 횟수와 최근 실행 시간이 갱신되는지 확인합니다.
