# Master App Platform (마스터 앱 플랫폼 관리 가이드)

본 플랫폼은 Firebase인프라를 기반으로 구축된 강력한 중앙 집중식 다중 서브 애플리케이션 통제 및 관리 시스템입니다.

## 🚀 STEP 7: 주요 핵심 기능 개요
- **사용자 대시보드(User Dashboard) 엔진:** 가입 승인(`approved`)을 취득한 일반 사용자들만을 위한 고유 워크스페이스 환경입니다.
- **실시간 비즈니스 로직 동기화:** 최고 관리자가 관리자 콘솔에서 특정 서브 애플리케이션의 토글 스위치를 활성화/비활성화함에 따라, 일반 사용자의 대시보드에 접근 가능 카드가 실시간으로 생성되거나 차단되는 안전한 격리 바인딩이 구현되었습니다.
- **강력한 상태 분기 및 예외 우회 방지:** 권한이 승인되지 않은 비정상 상태 유저(`pending`, `rejected`)는 클라이언트 라우팅 가드(`routing.js`)에 의해 대시보드 및 관리자 페이지 접근이 전면 차단됩니다.

## 🛠 설치 및 시작 가이드
1. 모든 프로젝트 소스 파일(`.html`, `.css`, `.js`, `.md`)을 동일한 루트 폴더 디렉토리에 위치시킵니다.
2. `firebase.js` 파일 내의 `firebaseConfig` 객체 안에 본인의 실제 Firebase Web SDK 프로젝트 키 값을 입력해 줍니다.
3. Live Server(VS Code 확장 프로그램 등) 혹은 자체 로컬 웹 서버 환경을 사용하여 `index.html` 파일을 구동해 줍니다.
4. 신규 유저로 이용 신청 후 관리자 계정(DB 상에서 `role: "admin"`으로 직접 수정)으로 전환하여 유저 승인 및 서브 애플리케이션 라우팅 기능을 마음껏 제어할 수 있습니다.
