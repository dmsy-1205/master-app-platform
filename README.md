# Master App Platform - STEP 8 앱 연결 v1

## 프로젝트 개요
Master App Platform은 여러 개의 웹 애플리케이션을 하나의 플랫폼에서 운영하기 위한 통합 플랫폼입니다.

현재 버전은 `master-app-platform-step08-v1` 입니다.

## 이번 버전에서 완료한 내용

### STEP 8 앱 연결 시스템
- 사용자 Dashboard에서 활성화된 서브 애플리케이션을 카드 형태로 표시하도록 확장했습니다.
- 각 앱 카드에 앱 이름 설명 버전 라우팅 경로 실행 방식을 표시합니다.
- 승인 완료 사용자만 앱 실행 버튼을 사용할 수 있도록 기존 STEP7 승인 상태와 연결했습니다.
- 앱 실행 시 Firebase에 실행 횟수와 최근 실행 시간을 기록합니다.
- 사용자별 실행 기록을 `appRunLogs/{uid}/{appId}` 경로에 저장합니다.
- 사용자 정보에는 마지막 실행 앱 정보를 `users/{uid}/lastAppLaunch`에 저장합니다.
- 앱 실행 방식은 세 가지를 지원합니다.
  - `router`: 플랫폼 내부 Runtime View에서 실행
  - `newTab`: 새 탭으로 외부 URL 실행
  - `sameTab`: 현재 창에서 외부 URL로 이동

### 관리자 앱 등록 확장
- STEP6 앱 등록 폼에 앱 버전 입력 필드를 추가했습니다.
- STEP6 앱 등록 폼에 실행 방식 선택 필드를 추가했습니다.
- 기존 등록 앱을 다시 동기화할 때 실행 횟수와 최근 실행 기록이 초기화되지 않도록 보존 처리했습니다.
- 관리자 라우팅 테이블에서 앱 버전과 실행 방식을 함께 확인할 수 있도록 표시를 확장했습니다.

### 라우터 보강
- 기존 MasterRouter 구조는 유지했습니다.
- 내부 앱은 기존처럼 `entryUrl`을 fetch하여 Runtime View에 삽입합니다.
- 외부 URL 앱은 실행 방식에 따라 새 탭 또는 현재 창으로 실행합니다.
- 비활성 앱은 기존처럼 라우팅 대상에서 제외됩니다.

## 테스트 방법
1. ZIP 파일 압축을 풉니다.
2. GitHub 저장소에 기존 프로젝트 파일을 덮어쓰기 업로드합니다.
3. Netlify 자동 배포가 완료될 때까지 기다립니다.
4. 로그인합니다.
5. 관리자 권한을 확인합니다.
6. STEP6 앱 등록 폼에서 앱 이름 경로 Entry URL 버전 실행 방식을 입력합니다.
7. 앱을 활성화 상태로 둡니다.
8. 승인 완료 사용자로 Dashboard에서 앱 실행 버튼을 누릅니다.
9. 내부 실행 앱은 Runtime View에 표시되는지 확인합니다.
10. 외부 URL 앱은 새 탭 또는 현재 창 이동 방식으로 실행되는지 확인합니다.
11. Dashboard 카드의 실행 횟수와 최근 실행 시간이 갱신되는지 확인합니다.
