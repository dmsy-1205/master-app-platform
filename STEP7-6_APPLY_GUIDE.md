# STEP7-6 Visual System & Mobile Navigation Recovery

## 목적
Galaxy Fold / 모바일 화면에서 상단 메뉴가 가로 스크롤 방식으로 노출되던 문제를 수정하고, 예전 버전의 회색 패널과 밝은 배경 위 흰색 글자 문제를 정리합니다.

## 적용 기준
- 기준 버전: STEP7-5 적용 후 앱 실행 정상 상태
- 적용 방식: modified-files-only
- Firebase / Auth / 앱 승인 / 앱 실행 / Rules 수정 없음

## 포함 파일
- index.html
- workspace.js
- assets/css/platform-shell.css

## 수정 요약
1. 모바일·Fold 상단 메뉴를 가로 스크롤 메뉴가 아닌 접힘형 메뉴로 복구
2. 메뉴 버튼을 `메뉴` 단일 버튼으로 정리
3. 메뉴 선택 시 자동으로 메뉴 닫힘 유지
4. 예전 버전에서 남은 회색 입력창/패널 색상 완화
5. 활동 기록, 공지사항, 고객지원, 앱 카드 등에서 흰 글자가 밝은 배경에 묻히는 문제 방지
6. 빈 `Dynamic Navigation` 개발용 영역은 내용이 없을 때 숨김 처리
7. 플랫폼 전체 배경을 더 밝고 트렌디한 glass/light gradient 톤으로 정리

## 검수 항목
### 모바일
- 상단 메뉴가 가로로 밀리지 않는지 확인
- 메뉴 버튼 클릭 시 세로 목록으로 펼쳐지는지 확인
- 메뉴 선택 후 자동으로 닫히는지 확인
- Home / 내 앱 / 앱 둘러보기 / 앱 신청 / 활동 기록 / 공지사항 / 고객지원 이동 확인

### Galaxy Fold
- 접은 화면과 펼친 화면에서 메뉴가 화면을 과도하게 차지하지 않는지 확인
- 활동 기록 글자가 정상적으로 보이는지 확인
- 앱 둘러보기 검색창이 비활성 회색처럼 보이지 않는지 확인

### PC
- 기존 좌측 내비게이션 또는 넓은 화면 구조가 깨지지 않는지 확인
- Admin Center 메뉴가 관리자에게만 보이는지 확인

## 주의
이번 STEP은 디자인/반응형 UX 수정입니다. Firebase Rules, security.js, 앱 실행 권한 로직은 수정하지 않았습니다.
