# Development Report - STEP8 v2

## 버전

`master-app-platform-step08-v2`

## 작업 목적

STEP8 v1 테스트 중 발견된 화면 구조 문제 로그인 유지 문제 관리자 권한 유지 문제를 수정하고 앱 실행 연결 기능을 사용자 Dashboard 중심으로 정리했습니다.

## 수정 전 문제

- 회원가입 로그인 로그인 상태 DB 테스트 관리자 권한 테스트 사용자 Dashboard 회원 신청 관리자 승인 앱 등록 라우팅 테이블 앱 실행 화면이 한 페이지에 모두 표시됨
- 창을 닫고 다시 들어가도 로그인 상태가 오래 유지되어 개발 테스트가 혼란스러움
- 관리자 권한을 부여해도 재로그인 후 일반 사용자처럼 보이는 현상이 발생할 수 있음
- STEP8 앱 실행 기능이 화면상 명확하게 분리되어 보이지 않음

## 수정 방향

- 로그인 상태에 따라 화면 노출 제어
- 관리자 권한에 따라 관리자 화면 노출 제어
- 개발 테스트 도구는 접기/펼치기 영역으로 분리
- Firebase Auth 세션 유지 방식을 브라우저 세션으로 조정
- 관리자 권한은 `admins/{uid}`와 `users/{uid}/role`을 함께 사용
- STEP8 앱 실행 기록 저장 구조 유지

## 30년차 코딩 작업자 관점 검수 결론

이번 수정은 기능 추가보다 구조 안정화가 핵심입니다.
STEP9 보안 규칙으로 넘어가기 전에 화면 분리 권한 판정 세션 유지 문제를 먼저 정리한 것은 맞는 순서입니다.

단 아직 완전한 멀티 페이지 구조는 아니며 단일 HTML 안에서 섹션 표시를 제어하는 중간 단계입니다.
현재 개발 방식에서는 기존 기능을 깨뜨리지 않으면서 구조를 개선하는 안전한 접근입니다.

## 테스트 우선순위

1. 로그인 전 화면 표시
2. 로그인 후 사용자 화면 표시
3. 관리자 등록 후 관리자 화면 표시
4. 로그아웃 후 재로그인 시 관리자 권한 유지
5. 앱 등록 및 활성화 비활성화
6. Dashboard 앱 실행
7. 실행 로그 저장
